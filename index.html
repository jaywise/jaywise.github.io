Project mock-up/proposal

For my project, I am aiming to create an interactive, user-facing dashboard summarizing user behavior. The audience will be learners accessing the PAPO web app, whose purpose is to empower English language learners in Brazil. The dashboard will consist of sliders and radio buttons that in turn morph graphic representations of data in "real time" (no refreshing necessary). For example, a user may be asked, "What is your goal for learning English?" The user will then select the reasons and the "weight" of each reason, e.g., 40% social, 25% professional advancement, 35% entertainment.

After each iteration, or learning cycle, with the app, the user will be asked to reflect upon the previously configured learning goals. To extend the above example, users will be prompted to tweak the proportionality of reasons for learning English based upon the most recent learning cycle and their new "station" in life and in the English learning process.

Finally, the dashboard will be responsive to data collected during the learning cycle of the app, utilizing patterns in user behavior to suggest ways in which learning goals may have changed. Within this culminating, metacognitive phase of each learning cycle, the app will display a tooltip or other light visual intervention when users navigate (scroll or hover) to each graph. The suggestion may sound something like: "We noticed you select learning resources that are more entertainment-minded. Is it possible that your learning goals have changed here? Take a minute to look at your previous goals and tweak them according to how you, as a learner, may have changed."

March 14

After returning from a user testing trip to Brazil, I have a better understanding of the parts of the interface that I can start building out now vs. the parts I will need to wait on. For the moment, I'm focusing on how to gather data for the learner profile dashboard. This means brushing up on HTML form elements (checkboxes, radio buttons, etc.) and prototyping a bit in CodePen with D3. My ultimate goal is to make a set of data visualizations that can be manipulated by the user, but which are initially informed by a survey about what kind of learner the user is. The best step for now is to get a handle on how to take data from a form and feed it into a pie graph/bar graph/histogram. I'll wait on the touch interaction later, as I'm not totally sure how to go about setting up touchscreen movements like pinchs or drags to change, say, the proportions in a pie graph.

The arc is not actually interactive at this stage; the values that appear are hard-coded into the JS. So, the next step will be to build the form into a series of values that can be fed into the graph. With the arc chart, this would look like generating an arc section for each modality ticked, and then having a way for learners to adjust the proportionality of each learning modality to fit their own learner profile. Ideally, this would be done with tactile, grab-and expand-motions. First, however, I have to insert the basic functionality of having the graph values reflect the form values, and be responsive each time those values are adjusted (even after the first "submit", should a learner want to modify his or her response). My plan is to explore highcharts some more, but also look more deeply into the interactive capabilities of d3.js

March 25

After working with some D3 tutorials and checking out the API more, I have a stroner understanding of the capabilities of D3 towards building interactive graphs and charts. The baked-in JQuery aspects do make for a conducive set of tools towards my purpose of generating graphical representations of students' meta-cognitive reflections. It have to further investigate how much it affords for manipulating in "real time" both with after-the-submit value tweaks (i.e., "Are you sure your are a 30% visual learner? In your most recent learning cycle, you utilized visual leanring tools more than 70% of the time. Adjust the graph according to your current understanding of who you're becoming as a learner.") and with pinch-and-expand touchscreen interactivity whose data can feed back into the original HTML form elements to reflect "real-time" changes.

That said, D3 might be a bit of overkill for some basic graphical representations; it is more suited to large, multi-faceted sets of data. I might leverage a simpler library like Highcharts (or even the HTML5 canvas element API, which I have been meaning to dig into more) for more lightweight interactive graphs. D3 could perhaps be better suited for "big picture", cumulative evaluations of learner journeys across multiple meta-cognitive check-in's.
